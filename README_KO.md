## FEMTOCAM이 뭔가요?

ESP32-CAM 보드 하나와 MicroSD 카드만으로 완성되는 WiFi 카메라 모듈입니다. 브라우저에서 바로 접속해서 실시간 스트리밍을 보고, 타임랩스를 녹화하고, 파일을 관리할 수 있습니다.

- **실시간 스트리밍** — MJPEG, QVGA / VGA / SVGA
- **타임랩스 녹화** — 시간당 / 분당 / 초당 자유 입력, 재생 FPS 별도 설정
- **연속 녹화** — 1~24fps, 녹화 시간 타이머, 전원 차단 보호
- **파일 관리** — 다중 선택, 일괄 다운로드/삭제, 이름 변경
- **WiFi 자동 설정** — 첫 부팅 시 AP 생성, 브라우저에서 설정, mDNS 지원
- **다크 테마 웹UI** — Mainsail 스타일, 영어/한국어 전환
- **외부 라이브러리 0개** — 아두이노 코어와 ESP-IDF만 사용

접속: `http://femtocam.local/` 또는 `http://192.168.4.1/`

## 잠깐, FEMTO가 뭔데?

**FEMTO**는 한국의 3D 프린팅 커뮤니티 [DCinside 3D프린팅 마이너갤러리](https://gall.dcinside.com/mgallery/board/lists/?id=3dprinting)에서 개최되는 초저가 프린터 자작 경연대회입니다.

한때 국내에서 "손도리 Pico"라는 이름으로 판매된 이지스레드(Easythreed) 프린터가 있었습니다. 너무나도 장난감 같은 퀄리티 탓에 커뮤니티에서는 어떤 프린터든 싸구려라는 말을 들으면 "그래도 Pico보다는 낫지"라고 할 정도로 밈이 되었죠.

FEMTO는 그 Pico를 오마주하면서 — **Pico보다 더 저렴하지만, 장난감이 아닌 제대로 된 프린터를 만들어보자**는 도전에서 출발한 대회입니다. 이름은 Pico(10⁻¹²)보다 더 작은 단위인 Femto(10⁻¹⁵)에서 따왔습니다.

### FEMTO 자매 프로젝트

FEMTO 대회를 위해 개발되었지만, 범용으로 사용할 수 있는 독립 모듈들입니다.

| 프로젝트 | 설명 | 플랫폼 | 상태 |
|----------|------|--------|------|
| **FEMTO Nano XY** | 초저가 자작 3D 프린터 (대회 출품작) | Marlin | 개발중 |
| **FEMTOCAM** | 스트리밍 & 타임랩스 카메라 (이 저장소) | ESP32-CAM | 공개 |
| **FEMTO Shaper** | 인풋 쉐이핑 독립 모듈 | ESP32-C3 + ADXL345 | 개발중 |

## 왜 만들었나요?

FEMTO 대회의 목표는 초저가 프린터입니다. 클리퍼(Klipper)를 쓰면 라즈베리파이 같은 SBC가 필요한데, 그 SBC 가격만으로도 대회 예산을 초과할 수 있습니다. 그래서 메인보드 하나로 돌아가는 **마를린(Marlin)**을 선택했습니다.

그런데 마를린에는 문제가 하나 있습니다 — 클리퍼+Mainsail 조합에서는 당연하게 쓰던 **카메라 스트리밍과 타임랩스 녹화**가 안 됩니다. SBC가 없으니 카메라를 물릴 곳이 없는 거죠. 3D 프린터에서 타임랩스를 찍고 싶다는 건 사치가 아니라, 출력 실패를 확인하고 결과물을 기록하는 실용적인 필요입니다.

그래서 **ESP32-CAM으로 독립 카메라 모듈**을 만들기로 했습니다.

ESP32-CAM에는 Espressif의 카메라 예제가 있긴 합니다. 하지만 써본 사람은 알 겁니다 — 설정 항목이 화면을 가득 채우고, UI는 투박하고, 녹화 같은 실용적인 기능은 없습니다. Mainsail 같은 3D 프린터 웹UI에 어울리는 디자인은 더더욱 아니었죠.

FEMTOCAM은 그래서:

1. **단순화** — 복잡한 설정을 정리하고, 설명이 필요한 항목에는 툴팁을 달았습니다
2. **타임랩스** — 캡처 속도와 재생 FPS를 분리해서 제대로 된 타임랩스를 만들 수 있게 했습니다
3. **파일 관리** — 녹화 파일을 브라우저에서 바로 다운로드, 삭제, 이름 변경할 수 있습니다
4. **디자인** — Mainsail에서 영감받은 다크 테마로, 프린터 옆에 띄워놔도 자연스럽습니다

외부 라이브러리 0개, 코드 1,100줄. SBC 없이, 마를린 프린터에서도 타임랩스를 찍을 수 있는 독립 모듈입니다.

## 주요 기능 상세

### 스트리밍
- MJPEG 실시간 스트리밍 (QVGA / VGA / SVGA)
- Raw socket 전송 + 4KB 청크로 안정적인 VGA 스트리밍
- 스트림 끊김 시 자동 재접속

### 녹화
- 캡처 속도: `[숫자] FPS / [초당·분당·시간당]` 자유 입력
- 재생 FPS: 캡처와 독립 설정 — 분당 1장 캡처 → 15fps 재생 = 완성된 타임랩스
- 녹화 시간 타이머 (0~600분, 0=무제한)
- AVI 중간 플러시 — 전원이 꺼져도 마지막 플러시까지 파일 보호
- NTP 기반 파일명 (`20260409_143022.avi`) 또는 순번 (`aprec_001.avi`)

### 파일 관리
- 체크박스로 다중 선택 → 일괄 다운로드 / 일괄 삭제
- 인라인 이름 변경 (클릭 → 입력 → Enter)
- 녹화 중인 파일은 삭제/다운로드 차단

### WiFi
- 첫 부팅: `FEMTOCAM` AP 자동 생성 (비밀번호 없음)
- 브라우저에서 WiFi 설정 → 재부팅 → mDNS 접속
- DHCP / 고정 IP 선택
- 연결 끊김 → 자동 재접속 → 실패 시 AP 폴백

### 설정 (4개 섹션)
- **Camera** — 밝기, 명암, 채도, 화질, 감도, 화이트밸런스, 좌우/상하 반전
- **Recording** — 재생 FPS, SD 최소 여유, 캡처 실패 한도, 중간 저장 주기
- **Log** — 시리얼 없이 웹에서 보는 실시간 시스템 로그
- **System** — 기기 이름 (로고 반영, mDNS 반영), 펌웨어 정보, 공장 초기화

모든 설정 항목에 "이게 뭔데?" 싶은 사람을 위한 한줄 설명이 포함되어 있습니다.

### 기기 이름 커스터마이즈
- Settings → System에서 변경
- 좌측 상단 로고에 즉시 반영
- mDNS 주소로 사용 (`http://이름.local/`)
- 이름에 "cam"이 포함되면 악센트 컬러로 강조

## 타임랩스 설정 예시

| 용도 | 캡처 속도 | 재생 FPS | 결과 |
|------|----------|---------|------|
| 장기 촬영 (건축 등) | 1장 / 시간당 | 15fps | 하루 = 1.6초 |
| 3D 프린트 타임랩스 | 1장 / 분당 | 15fps | 1시간 = 4초 |
| 빠른 타임랩스 | 1장 / 초당 | 15fps | 15배속 |
| 실시간 녹화 | 15fps / 초당 | 15fps | 원속 재생 |

## 알려진 제한사항

- VGA/SVGA 스트리밍은 WiFi 환경에 따라 ~7fps로 제한될 수 있음
- 동시 스트리밍 1클라이언트
- NTP 시간은 STA 모드에서만 (AP 모드는 순번 파일명)
- ESP32 성능 한계로 실시간 워터마크 미지원
- WiFi가 불안정할 경우 도킹보드에서 분리 후 별도 5V 공급 권장

## 개발 이야기

이 프로젝트는 비전공자가 **Claude AI와의 바이브 코딩(Vibe Coding)**으로 만들었습니다. 아키텍처 설계, 펌웨어, 웹UI, 하드웨어 디버깅까지 전 과정을 AI와 대화하며 개발했습니다.

자동화된 버그 테스트(10라운드, 100+항목)를 거쳤고 실제 ESP32-CAM에서 검증했지만, 예상치 못한 버그가 있을 수 있습니다. 이슈를 발견하면 편하게 알려주세요.
